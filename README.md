# Приложение для транскрибации

Docker-приложение для расшифровки аудио и видео с помощью `faster-whisper`. Модели кэшируются в отдельном volume, поэтому после пересборки контейнера их не нужно скачивать заново.

## Требования

- Docker
- NVIDIA Container Toolkit для запуска на GPU
- Токен Telegram-бота, если нужна расшифровка голосовых сообщений

## Быстрый старт

```bash
docker compose up --build
# Откройте http://localhost:8000
```

Для работы на GPU нужен Docker с настроенным NVIDIA Container Toolkit.

**Telegram-бот:** укажите переменную окружения `TELEGRAM_BOT_TOKEN`, чтобы включить бота. После этого можно отправлять ему голосовые сообщения, а он будет отвечать позицией в очереди и готовой транскрибацией.

## Кэширование моделей

Volume `model_cache` сохраняет:

- модели Whisper;
- кэш Hugging Face;
- кэш Torch и CTranslate2.

### Как происходит первое скачивание модели

1. Запустите приложение: `docker compose up --build`.
2. Загрузите файл или отправьте ссылку и нажмите `Transcribe`.
3. При первом запуске модель `large-v2` будет скачана в `/cache`.
4. При следующих запусках тот же volume будет использоваться повторно, поэтому модель загрузится с диска.
5. После `docker compose up --build` образ пересобирается, но volume `model_cache` сохраняется.

Чтобы заранее прогреть кэш, можно после первого запуска загрузить короткий аудиофайл. Скачивание модели начнется только при первой транскрибации.

### Кэш сборки pip

Папка `pip-cache/` используется как bind mount во время сборки, чтобы скачанные зависимости сохранялись на машине. Первый билд может скачать несколько гигабайт, а последующие сборки будут использовать локальный кэш.

## API

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/transcribe/upload` | `POST` | Загрузка файла через multipart: `file`, `language` |
| `/api/transcribe/url` | `POST` | Отправка ссылки через form: `url`, `language`; извлекается только аудио |
| `/api/status/{job_id}` | `GET` | Проверка статуса задачи |
| `/api/result/{job_id}` | `GET` | Получение результата транскрибации |

## Telegram-бот

Если задан `TELEGRAM_BOT_TOKEN`, приложение запускает Telegram-бота в том же процессе, что и FastAPI. Бот принимает голосовые и аудиосообщения, добавляет их в общую очередь и отправляет результат после обработки.

## Возможные проблемы

### Модель не скачивается

Проверьте, что у контейнера есть доступ к интернету и что volume `/cache` доступен для записи.

### GPU не используется

Проверьте установку NVIDIA Container Toolkit и доступность GPU внутри Docker. Для CPU-запуска нужно изменить Dockerfile и убрать GPU-секцию из `docker-compose.yml`.

## Запуск без GPU

Для CPU-режима можно заменить базовый образ и убрать настройки GPU:

```yaml
# В Dockerfile заменить первую строку на:
FROM python:3.11-slim

# Добавить установку ffmpeg:
RUN apt-get update && apt-get install -y ffmpeg && rm -rf /var/lib/apt/lists/*

# В docker-compose.yml удалить блок deploy.resources
```
